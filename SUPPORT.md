# Support

Thanks for using Erfana. This page explains where to go for each kind of help so
your request reaches the right place quickly.

## Questions, ideas, and usage help

Use [GitHub Discussions](https://github.com/qodeca/erfana/discussions) for
questions, how-to help, feature ideas, and general conversation about using
Erfana. It is the best place to start when you are not sure something is a bug.

## Bug reports and feature requests

Open an issue with one of the templates:

- **Bug report** – [`bug_report.yml`](.github/ISSUE_TEMPLATE/bug_report.yml):
  something is broken or behaving unexpectedly. Include your Erfana version
  (Help → About), your OS, and clear reproduction steps.
- **Feature request** –
  [`feature_request.yml`](.github/ISSUE_TEMPLATE/feature_request.yml): propose a
  new capability or improvement.

Start a new issue from the repository's **Issues** tab and pick the matching
template.

## Security vulnerabilities

**Do not** report security issues through public issues or Discussions. Follow
the private process in [`SECURITY.md`](SECURITY.md) – preferably GitHub's
[private vulnerability reporting](https://github.com/qodeca/erfana/security/advisories/new)
form. See [`SECURITY.md`](SECURITY.md) for scope and disclosure details.

## Supported versions

Erfana is at a `0.x` stage. Only the latest released version receives fixes,
including security fixes. Please reproduce issues on the latest release before
reporting.

## Contributing

If you would like to help rather than ask for help, see
[`CONTRIBUTING.md`](CONTRIBUTING.md) for the contributor workflow and quality
gates.
